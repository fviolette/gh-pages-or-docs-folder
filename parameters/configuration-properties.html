Configuration properties

The DITA-OT uses .properties files and internal properties that store configuration settings for the toolkit and its plug-ins. Configuration properties are available to both Ant and Java processes, but unlike argument properties, they cannot be set at run time.

  • plugin.properties file
    The plugin.properties file is used to store configuration properties that are set by the plug-in installation process. The file is located in the lib/org.dita.dost.platform directory; it is regenerated each time the installation process is run and so should not be edited manually.
  • configuration.properties file
    The lib/configuration.properties file controls certain common properties, as well as some properties that control PDF processing.
